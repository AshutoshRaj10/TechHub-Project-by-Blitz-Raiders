# Contributing to DB Analyser Bot

First off, thanks for taking the time to contribute! 🎉

The following is a set of guidelines for contributing to **DB Analyser Bot**, developed by **Team Blitz Raiders**. These are mostly guidelines, not rules. Use your best judgment, and feel free to propose changes to this document in a pull request.

## How Can I Contribute?

### Reporting Bugs
* Check the existing issues to see if the bug has already been reported.
* If you find a new bug, please open an issue with a clear title and description, including steps to reproduce the bug and your environment details.

### Suggesting Enhancements
* Open an issue to discuss the enhancement before starting any work.
* Provide context on why this enhancement would be beneficial.

### Pull Requests
1. **Fork the repo** and create your branch from `main`.
2. **If you've added code** that should be tested, add tests.
3. **If you've changed APIs**, update the documentation.
4. **Ensure the test suite passes**.
5. **Format your code** according to project standards.
6. **Submit a pull request!**

## Development Workflow

### Backend
* Use `FastAPI` best practices.
* Keep the `agent.py` logic modular.
* Ensure SQL generation prompts are tested for edge cases.

### Frontend
* Use functional components and hooks.
* Maintain the design aesthetic with Tailwind CSS.
* Test new visualizations across different screen sizes.

## Style Guide
* **Python**: PEP 8
* **JavaScript/React**: StandardJS or Prettier defaults
* **Git**: Use descriptive commit messages (e.g., `feat: add support for PostgreSQL`)

Thank you for contributing to the future of AI-driven data intelligence!
